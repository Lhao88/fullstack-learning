# 接口说明

## 基础信息

- 基础地址：`http://localhost:3000`
- Swagger：`http://localhost:3000/api-docs`
- 请求体默认使用 JSON。
- 上传接口使用 `multipart/form-data`。

除注册、登录、刷新 token、退出登录外，其他业务接口都需要请求头：

```text
Authorization: Bearer accessToken
```

统一错误响应格式：

```json
{
  "statusCode": 401,
  "message": "Unauthorized",
  "path": "/task",
  "timestamp": "2026-07-01T00:00:00.000Z"
}
```

## Postman 推荐流程

1. `POST /auth/register` 注册普通用户。
2. `POST /auth/login` 登录。
3. 复制 `data.accessToken`。
4. 在受保护接口 Headers 中加入 `Authorization: Bearer accessToken`。
5. access token 过期后，用 `POST /auth/refresh` 换新 token。
6. 退出登录时调用 `POST /auth/logout` 让 refresh token 失效。

## Auth 接口

### 注册

- 方法：`POST`
- 路径：`/auth/register`
- 权限：无需登录

请求体：

```json
{
  "email": "user@example.com",
  "password": "123456",
  "nickname": "小豪"
}
```

响应：

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": "user-id",
    "email": "user@example.com",
    "nickname": "小豪",
    "role": "user",
    "status": "enabled",
    "avatarUrl": null,
    "createdAt": "2026-07-01T00:00:00.000Z",
    "updatedAt": "2026-07-01T00:00:00.000Z"
  }
}
```

### 登录

- 方法：`POST`
- 路径：`/auth/login`
- 权限：无需登录

请求体：

```json
{
  "email": "user@example.com",
  "password": "123456"
}
```

响应：

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "accessToken": "xxx",
    "refreshToken": "yyy",
    "user": {
      "id": "user-id",
      "email": "user@example.com",
      "nickname": "小豪",
      "role": "user",
      "status": "enabled",
      "avatarUrl": null,
      "createdAt": "2026-07-01T00:00:00.000Z",
      "updatedAt": "2026-07-01T00:00:00.000Z"
    }
  }
}
```

说明：

- `accessToken` 用于访问受保护接口。
- `refreshToken` 用于刷新登录态。
- 数据库只保存 refresh token 的 hash，不保存明文 refresh token。
- 禁用用户不能登录。

### 刷新 Token

- 方法：`POST`
- 路径：`/auth/refresh`
- 权限：无需 access token，需要 refresh token

请求体：

```json
{
  "refreshToken": "yyy"
}
```

响应：

```json
{
  "code": 200,
  "message": "刷新 token 成功",
  "data": {
    "accessToken": "new-access-token",
    "refreshToken": "new-refresh-token",
    "user": {}
  }
}
```

说明：

- refresh 成功后旧 refresh token 会失效。
- 前端应保存新的 access token 和 refresh token。

### 退出登录

- 方法：`POST`
- 路径：`/auth/logout`
- 权限：无需 access token，需要 refresh token

请求体：

```json
{
  "refreshToken": "yyy"
}
```

响应：

```json
{
  "code": 200,
  "message": "退出登录成功"
}
```

### 获取当前用户

- 方法：`GET`
- 路径：`/auth/me`
- 权限：需要登录

响应：

```json
{
  "code": 200,
  "data": {
    "id": "user-id",
    "email": "user@example.com",
    "nickname": "小豪",
    "role": "user",
    "status": "enabled",
    "avatarUrl": "/uploads/avatar.png",
    "createdAt": "2026-07-01T00:00:00.000Z",
    "updatedAt": "2026-07-01T00:00:00.000Z"
  }
}
```

### 上传头像

- 方法：`POST`
- 路径：`/auth/avatar`
- 权限：需要登录
- Content-Type：`multipart/form-data`

表单字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| file | file | 是 | png、jpg、webp 图片，最大 2MB |

响应：

```json
{
  "code": 200,
  "message": "头像上传成功",
  "data": {
    "id": "user-id",
    "avatarUrl": "/uploads/filename.png"
  }
}
```

## Task 接口

任务接口都需要登录，并且只能操作当前用户自己的任务。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/task` | 获取当前用户任务列表 |
| GET | `/task/:id` | 获取任务详情 |
| POST | `/task` | 创建任务 |
| PATCH | `/task/:id` | 更新任务 |
| DELETE | `/task/:id` | 删除任务 |
| PATCH | `/task/:id/status/next` | 流转到下一个状态 |

创建任务请求体：

```json
{
  "title": "完成接口联调",
  "description": "将前端任务列表改为调用后端接口",
  "level": "high",
  "categoryId": "category-id"
}
```

更新任务请求体：

```json
{
  "title": "更新后的标题",
  "description": "更新后的描述",
  "status": "doing",
  "level": "medium",
  "categoryId": null
}
```

状态流转规则：

```text
todo -> doing -> done -> todo
```

## Category 接口

分类接口都需要登录，并且只能操作当前用户自己的分类。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/category` | 获取分类列表 |
| POST | `/category` | 创建分类 |
| PATCH | `/category/:id` | 更新分类 |
| DELETE | `/category/:id` | 删除分类 |

创建分类请求体：

```json
{
  "name": "学习",
  "color": "blue"
}
```

说明：

- 同一用户下分类名称不能重复。
- 删除分类不会删除任务，任务会变成未分类。

## Upload 接口

### 上传图片

- 方法：`POST`
- 路径：`/upload/image`
- 权限：需要登录
- Content-Type：`multipart/form-data`

表单字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| file | file | 是 | png、jpg、webp 图片，最大 2MB |

响应：

```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "filename": "filename.png",
    "url": "/uploads/filename.png",
    "mimetype": "image/png",
    "size": 1024
  }
}
```

## Admin 接口

管理员接口需要登录，且当前用户必须是 `super_admin`。

普通用户访问会返回 `403`。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/admin/users` | 获取用户列表 |
| GET | `/admin/users/:id` | 获取用户详情 |
| PATCH | `/admin/users/:id/status` | 启用或禁用用户 |
| GET | `/admin/categories` | 获取全部用户的分类列表 |
| POST | `/admin/categories` | 为指定用户创建分类 |
| PATCH | `/admin/categories/:id` | 更新分类 |
| DELETE | `/admin/categories/:id` | 删除分类 |

更新用户状态请求体：

```json
{
  "status": "disabled"
}
```

可选状态：

- `enabled`
- `disabled`

说明：

- 禁用用户后，该用户不能登录。
- 已登录用户被禁用后，再访问受保护接口也会失败。
- 当前实现禁止管理员禁用自己的账号。

创建管理员分类请求体：

```json
{
  "userId": "user-id",
  "name": "项目",
  "color": "purple"
}
```

更新管理员分类请求体：

```json
{
  "userId": "user-id",
  "name": "文档",
  "color": "green"
}
```

说明：

- 管理员分类接口可以查看和管理全部用户的分类。
- 分类仍然属于某个具体用户。
- 同一个用户下分类名称不能重复。
- 删除分类后，相关任务会变成未分类。

## 参数校验

| 字段 | 规则 |
| --- | --- |
| email | 必填，必须是邮箱格式 |
| password | 必填，注册时至少 6 个字符 |
| title | 必填，不能为空 |
| description | 必填，不能为空 |
| level | `high`、`medium`、`low` |
| status | 任务状态：`todo`、`doing`、`done` |
| categoryId | 可选，必须属于当前用户 |
| category.name | 必填，同一用户下唯一，最多 20 个字符 |
| user.status | `enabled`、`disabled` |
