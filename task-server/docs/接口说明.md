# 任务管理接口说明

## 基础说明

当前第二周后端先使用内存数组保存任务数据，后续第三周再接入数据库。

本接口文档先按任务管理核心流程设计，包括任务列表、详情、新增、更新、删除，以及任务状态流转。

## 基础地址

```text
http://localhost:3000
```

## 数据结构

### TaskItem

```ts
interface TaskItem {
  id: string
  title: string
  description: string
  status: 'todo' | 'in-progress' | 'done'
  level: 'high' | 'medium' | 'low'
  createdAt: string
  updatedAt: string
}
```

说明：

- `status`：任务状态。
  - `todo`：待处理
  - `in-progress`：进行中
  - `done`：已完成
- `level`：任务优先级。
  - `high`：高
  - `medium`：中
  - `low`：低
- `createdAt` 和 `updatedAt` 在接口响应中使用字符串格式，例如 ISO 时间字符串。

## 3. 创建任务

### 请求方法

```text
POST
```

### 请求路径

```text
/task
```

### 请求参数

无。

### 请求体

| 字段名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| title | string | 是 | 任务标题 |
| description | string | 是 | 任务描述 |
| level | high \| medium \| low | 是 | 任务优先级 |

示例：

```json
{
  "title": "学习 NestJS Controller",
  "description": "理解 Controller 如何接收请求并调用 Service",
  "level": "high"
}
```

### 响应数据

```json
{
  "code": 200,
  "data": {
    "id": "task-2",
    "title": "学习 NestJS Controller",
    "description": "理解 Controller 如何接收请求并调用 Service",
    "status": "todo",
    "level": "high",
    "createdAt": "2026-06-26T08:30:00.000Z",
    "updatedAt": "2026-06-26T08:30:00.000Z"
  }
}
```

说明：

- 新建任务默认 `status` 为 `todo`。
- `id`、`createdAt`、`updatedAt` 由后端生成。

## 4. 更新任务

### 请求方法

```text
PATCH
```

### 请求路径

```text
/task/:id
```

### 请求参数

| 参数名 | 位置 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| id | path | string | 是 | 任务 ID |

### 请求体

| 字段名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| title | string | 否 | 任务标题 |
| description | string | 否 | 任务描述 |
| status | todo \| in-progress \| done | 否 | 任务状态 |
| level | high \| medium \| low | 否 | 任务优先级 |

示例：

```json
{
  "title": "学习 NestJS Service",
  "description": "把任务业务逻辑放到 Service 中",
  "level": "medium"
}
```

### 响应数据

```json
{
  "code": 200,
  "message": "更新成功",
  "data": {
    "id": "task-2",
    "title": "学习 NestJS Service",
    "description": "把任务业务逻辑放到 Service 中",
    "status": "todo",
    "level": "medium",
    "createdAt": "2026-06-26T08:30:00.000Z",
    "updatedAt": "2026-06-26T09:00:00.000Z"
  }
}
```

说明：

- 更新任务时只需要传要修改的字段。
- 每次更新后，后端需要刷新 `updatedAt`。

### 错误响应

任务不存在时返回：

```json
{
  "statusCode": 404,
  "message": "任务不存在",
  "error": "Not Found"
}
```

## 6. 流转到下一个状态

### 请求方法

```text
PATCH
```

### 请求路径

```text
/task/:id/status/next
```

### 请求参数

| 参数名 | 位置 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| id | path | string | 是 | 任务 ID |

### 请求体

无。

### 响应数据

```json
{
  "code": 200,
  "message": "状态更新成功",
  "data": {
    "id": "task-1",
    "title": "完成任务管理页面",
    "description": "实现任务列表、筛选和状态切换",
    "status": "in-progress",
    "level": "high",
    "createdAt": "2026-06-26T08:00:00.000Z",
    "updatedAt": "2026-06-26T09:10:00.000Z"
  }
}
```

说明：

- 根据任务当前状态，自动流转到下一个状态。
- 状态流转规则：
  - `todo` -> `in-progress`
  - `in-progress` -> `done`
  - `done` -> `todo`
- 修改后刷新 `updatedAt`。

### 错误响应

任务不存在时返回：

```json
{
  "statusCode": 404,
  "message": "任务不存在",
  "error": "Not Found"
}
```

## 参数校验规则

当前 DTO 和 `ValidationPipe` 按下面规则校验：

| 字段名 | 校验规则 |
| --- | --- |
| title | 必填，字符串，不能为空 |
| description | 必填，字符串，不能为空 |
| level | 必须是 `high`、`medium`、`low` 之一 |
| status | 必须是 `todo`、`in-progress`、`done` 之一 |

参数错误时返回：

```json
{
  "statusCode": 400,
  "message": [
    "title should not be empty"
  ],
  "error": "Bad Request"
}
```
