# 数据库说明

## 基础信息

当前后端使用：

- 数据库：PostgreSQL
- ORM：Prisma
- 数据库配置文件：`prisma/schema.prisma`
- Prisma 配置文件：`prisma.config.ts`
- 迁移目录：`prisma/migrations`
- 开发种子数据：`prisma/seed-dev.sql`

数据库连接字符串通过 `task-server/.env` 中的 `DATABASE_URL` 配置，不要把真实连接字符串提交到仓库。

## 当前模型概览

当前数据库包含两个核心模型：

- `User`：用户表，用于注册、登录、JWT 当前用户识别。
- `Task`：任务表，用于保存任务标题、描述、状态、优先级和所属用户。

关系：

```text
User 1 --- n Task
```

含义：

- 一个用户可以拥有多条任务。
- 一条任务只属于一个用户。
- 删除用户时，该用户下面的任务会一起删除。

## User 表

Prisma 模型：

```prisma
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  password  String
  nickname  String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  tasks     Task[]
}
```

字段说明：

| 字段名 | 数据库类型 | Prisma 类型 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| id | TEXT | String | 主键，默认 uuid | 用户 ID |
| email | TEXT | String | 唯一 | 用户邮箱，用于登录 |
| password | TEXT | String | 必填 | 加密后的密码 hash，不保存明文密码 |
| nickname | TEXT | String? | 可为空 | 用户昵称 |
| createdAt | TIMESTAMP(3) | DateTime | 默认当前时间 | 创建时间 |
| updatedAt | TIMESTAMP(3) | DateTime | 自动更新 | 更新时间 |
| tasks | - | Task[] | 一对多关系 | 当前用户拥有的任务列表 |

注意：

- `email` 有唯一索引，重复注册会失败。
- `password` 保存的是 bcrypt hash。
- 接口响应中不应该返回 `password` 字段。

## Task 表

Prisma 模型：

```prisma
model Task {
  id          String     @id @default(uuid())
  title       String
  description String
  status      TaskStatus @default(todo)
  level       TaskLevel  @default(medium)
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt

  userId      String
  user        User       @relation(fields: [userId], references: [id], onDelete: Cascade)
}
```

字段说明：

| 字段名 | 数据库类型 | Prisma 类型 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| id | TEXT | String | 主键，默认 uuid | 任务 ID |
| title | TEXT | String | 必填 | 任务标题 |
| description | TEXT | String | 必填 | 任务描述 |
| status | TaskStatus | TaskStatus | 默认 `todo` | 任务状态 |
| level | TaskLevel | TaskLevel | 默认 `medium` | 任务优先级 |
| createdAt | TIMESTAMP(3) | DateTime | 默认当前时间 | 创建时间 |
| updatedAt | TIMESTAMP(3) | DateTime | 自动更新 | 更新时间 |
| userId | TEXT | String | 外键 | 任务所属用户 ID |
| user | - | User | 多对一关系 | 任务所属用户 |

关系约束：

```prisma
user User @relation(fields: [userId], references: [id], onDelete: Cascade)
```

说明：

- `Task.userId` 关联 `User.id`。
- `onDelete: Cascade` 表示删除用户时，该用户的任务会自动删除。
- 任务接口中不应该由前端传 `userId`，而是根据 JWT 当前用户自动绑定。

## TaskStatus 枚举

Prisma 定义：

```prisma
enum TaskStatus {
  todo
  doing
  done
}
```

字段含义：

| 值 | 中文含义 | 说明 |
| --- | --- | --- |
| todo | 待处理 | 新建任务默认状态 |
| doing | 进行中 | 任务已经开始 |
| done | 已完成 | 任务完成或归档 |

当前状态流转规则：

```text
todo -> doing -> done -> todo
```

对应接口：

```text
PATCH /task/:id/status/next
```

## TaskLevel 枚举

Prisma 定义：

```prisma
enum TaskLevel {
  high
  medium
  low
}
```

字段含义：

| 值 | 中文含义 | 说明 |
| --- | --- | --- |
| high | 高 | 高优先级任务 |
| medium | 中 | 默认优先级 |
| low | 低 | 低优先级任务 |

## 迁移说明

当前已经存在初始化迁移：

```text
prisma/migrations/20260627151008_init_user_task/migration.sql
```

该迁移创建了：

- `TaskStatus` 枚举
- `TaskLevel` 枚举
- `User` 表
- `Task` 表
- `User.email` 唯一索引
- `Task.userId -> User.id` 外键关系

### 创建迁移

在 `task-server` 目录下执行：

```cmd
npx prisma migrate dev --name init_user_task
```

说明：

- 根据 `prisma/schema.prisma` 的模型定义生成迁移 SQL。
- 把迁移应用到本地开发数据库。
- 会在 `prisma/migrations` 下生成迁移记录。

后续如果新增分类、标签、统计相关表，可以使用新的迁移名称，例如：

```cmd
npx prisma migrate dev --name add_category_tag
```

### 生成 Prisma Client

修改 `schema.prisma` 后，通常需要执行：

```cmd
npx prisma generate
```

说明：

- 根据最新 Prisma 模型生成 TypeScript 客户端。
- 后端通过 `@prisma/client` 获得类型和数据库操作 API。

## Seed 说明

当前开发 seed 文件：

```text
prisma/seed-dev.sql
```

作用：

- 创建一个开发测试用户。
- 创建 20 条开发任务。
- 所有 seed 任务都挂到开发测试用户下面。
- 使用 `ON CONFLICT`，可以重复执行，不会重复插入同一批 seed 数据。

在 `task-server` 目录下执行：

```cmd
npx prisma db execute --file prisma/seed-dev.sql
```

说明：

- 该命令直接执行 SQL 文件。
- 执行前需要确保 `.env` 中的 `DATABASE_URL` 指向本地开发数据库。
- seed 文件中的用户密码字段保存的是 hash，不应该写入明文密码。

## 第四周扩展参考

后续如果要扩展分类、标签、统计，可以基于当前结构继续增加模型。

### 分类

一个任务通常只属于一个分类时，可以设计：

```text
User 1 --- n Category
Category 1 --- n Task
```

可能新增字段：

```text
Task.categoryId
```

### 标签

一个任务可以有多个标签，一个标签也可以属于多个任务时，可以设计多对多关系：

```text
Task n --- n Tag
```

Prisma 可以用隐式多对多，也可以单独建中间表。

### 统计

统计通常不一定要新建表，优先可以基于 `Task` 表查询计算：

- 按状态统计任务数量。
- 按优先级统计任务数量。
- 按 `updatedAt` 统计本周完成任务。
- 按用户统计个人任务概览。

如果后续需要保存历史统计快照，再考虑新增统计表。
