# 数据库说明

## 基础信息

- 数据库：PostgreSQL
- ORM：Prisma
- 模型文件：`prisma/schema.prisma`
- 迁移目录：`prisma/migrations`
- 开发 seed：`prisma/seed-dev.sql`

数据库连接通过环境变量 `DATABASE_URL` 配置，不要把真实连接字符串提交到 Git。

## 模型关系

```text
User 1 --- n Task
User 1 --- n Category
User 1 --- n RefreshToken
Category 1 --- n Task
```

说明：

- 一个用户可以有多条任务。
- 一个用户可以有多个分类。
- 一个用户可以有多个 refresh token 记录。
- 一条任务只属于一个用户。
- 一条任务可以属于一个分类，也可以未分类。
- 删除用户时，会级联删除该用户的任务、分类和 refresh token。
- 删除分类时，不删除任务，任务的 `categoryId` 会变成 `null`。

## User 表

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | String | 主键，默认 uuid |
| email | String | 唯一，用于登录 |
| password | String | bcrypt hash，不保存明文密码 |
| nickname | String? | 用户昵称 |
| role | UserRole | 用户角色，默认 `user` |
| status | UserStatus | 用户状态，默认 `enabled` |
| avatarUrl | String? | 头像访问地址 |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

角色：

```prisma
enum UserRole {
  user
  super_admin
}
```

状态：

```prisma
enum UserStatus {
  enabled
  disabled
}
```

说明：

- `user` 是普通用户，只能管理自己的任务、分类和头像。
- `super_admin` 是超级管理员，可以访问管理后台。
- `disabled` 用户不能登录，也不能继续访问受保护接口。
- 头像只保存 URL，不把图片二进制直接存进数据库。

## RefreshToken 表

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | String | 主键，默认 uuid，同时作为 refresh token 的 tokenId |
| tokenHash | String | refresh token 的 bcrypt hash |
| expiresAt | DateTime | 过期时间 |
| createdAt | DateTime | 创建时间 |
| userId | String | 所属用户 ID |

说明：

- 登录成功会生成 access token 和 refresh token。
- refresh token 明文只返回给前端一次，数据库只保存 hash。
- 刷新 token 成功后，会删除旧 refresh token，并签发新的 refresh token。
- 退出登录会删除对应 refresh token 记录。

## Category 表

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | String | 主键，默认 uuid |
| name | String | 分类名称 |
| color | String | 分类颜色，默认 `blue` |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |
| userId | String | 所属用户 ID |

约束：

```prisma
@@unique([userId, name])
```

说明：

- 分类按用户隔离。
- 同一用户下分类名称不能重复。

## Task 表

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | String | 主键，默认 uuid |
| title | String | 任务标题 |
| description | String | 任务描述 |
| status | TaskStatus | 任务状态，默认 `todo` |
| level | TaskLevel | 任务优先级，默认 `medium` |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |
| userId | String | 所属用户 ID |
| categoryId | String? | 所属分类 ID，可为空 |

任务状态：

```prisma
enum TaskStatus {
  todo
  doing
  done
}
```

状态流转：

```text
todo -> doing -> done -> todo
```

任务优先级：

```prisma
enum TaskLevel {
  high
  medium
  low
}
```


## Seed

开发 seed 文件：

```text
prisma/seed-dev.sql
```

执行：

```cmd
cd /d E:\vscode\fullstack\task-server
npx prisma db execute --file prisma/seed-dev.sql
```

seed 当前用于准备：

- 普通测试用户。
- 超级管理员用户。
- 测试分类。
- 测试任务。

注意：

- seed 中的密码字段应保存 hash。
- seed 可以用于本地开发，不应该直接当作生产数据。
