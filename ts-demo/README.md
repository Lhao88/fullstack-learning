# TypeScript 任务管理练习

这是一个基于 React、TypeScript、Vite 和 Ant Design 实现的任务管理练习项目。

项目来自全栈与 App 开发学习路线的第一周练习，目标是通过一个小型任务管理页面，熟悉 React 组件拆分、TypeScript 类型定义、列表渲染、条件渲染、表单弹窗、状态管理和基础业务交互。

## 技术栈

- React 19
- TypeScript
- Vite
- Ant Design
- Zustand

## 已完成功能

### 页面结构

- 工作台页面：展示任务统计和三列任务看板。
- 任务列表页面：展示完整任务表格，支持搜索、筛选、新增、编辑、删除。
- 归档记录页面：展示已完成任务，支持查看详情和恢复任务。

### 任务看板

- 按任务状态分为待处理、进行中、已完成三列。
- 每列展示对应状态的任务数量。
- 任务卡片显示标题、描述、优先级和更新时间。
- 优先级支持高、中、低三种展示。
- 任务卡片按优先级从高到低排序，同优先级按更新时间倒序排序。
- 更新时间如果是当天，显示“今天”，否则显示具体日期。
- 待处理任务可以点击“开始”，进行中任务可以点击“完成”。

### 任务列表

- 表格数据来自统一任务状态，不再使用页面内写死数据。
- 支持按标题和描述进行关键词搜索。
- 支持按任务状态筛选。
- 支持按任务优先级筛选。
- 支持重置全部筛选条件。
- 表格滚动时表头保持固定。
- 根据任务状态动态显示操作按钮：
  - 待处理：显示“开始”
  - 进行中：显示“完成”
  - 已完成：不显示状态流转按钮

### 新增、编辑、删除

- 支持新增任务弹窗。
- 支持编辑任务弹窗，并回填当前任务数据。
- 新增和编辑复用同一个 `TaskModal` 组件。
- 删除任务前使用确认弹窗，确认后再删除。
- 新增、编辑、删除后，工作台、任务列表和归档记录会同步更新。

### 归档记录

- 自动筛选状态为已完成的任务作为归档任务。
- 统计已完成任务数量。
- 统计本周归档任务数量。
- 支持查看已完成任务详情。
- 支持把已完成任务恢复为进行中。

### 状态管理

- 使用 Zustand 管理任务列表状态。
- 当前 store 提供：
  - `tasks`
  - `addTask`
  - `updateTask`
  - `removeTask`

## 本地运行

安装依赖：

```cmd
npm install
```

启动开发服务器：

```cmd
npm run dev
```

构建项目：

```cmd
npm run build
```

预览构建结果：

```cmd
npm run preview
```

## 第一周学习重点

- 练习 React 页面组件和通用组件拆分。
- 练习使用 TypeScript 定义业务数据类型。
- 练习根据状态和优先级进行条件渲染。
- 练习列表渲染、筛选和排序。
- 练习 Ant Design 的表格、卡片、表单、弹窗、确认框等组件。
- 练习用 Zustand 管理跨页面共享状态。
- 练习把业务逻辑拆分到更合适的页面和组件中。
